# Архитектура

- Доменный слой: АТД
- Прикладной слой: игровой цикл, состояния, команды
- Слой инфраструктуры: консольный ввод-вывод, запись статистики

## Инварианты

- Статистика и журнал ходов обновляются только после успешного игрового действия
- Только одно активное состояние игры в любой момент времени

## Схема создания объектов

- Program.cs как Composition Root, инициализация сервисов в нём
- GameSessionATD как оркестратор игры

# Тесты для типовых сценариев

- Образование комбинации после перестановки двух соседних клеток
- Перестановка двух соседних клеток без комбинации и её откат в изначальное состояние
- Поле успешно заполнено после удаления всех образовавшихся после хода комбинаций
- Активация бонуса, прохождение его через цикл игровой логике, итоговое состояние корректно
- При отсутствии возможных ходов игра перезодит в состояние GameOver
